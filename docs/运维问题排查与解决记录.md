## 1. 问题概述

本次运维工作中遇到以下问题：

1. **HDFS 配置错误**: `hdfs-site.xml` 存在 XML 语法错误，导致 HDFS 命令无法执行
2. **YARN 端口绑定**: ResourceManager 和 JobHistoryServer 只绑定到特定 IP，无法通过 localhost 访问
3. **Grafana 端口绑定**: Grafana 默认只监听 IPv6，需要配置为监听所有接口

## 2. HDFS 配置问题

### 2.1 问题现象

执行 HDFS 命令时报错：
```
com.ctc.wstx.exc.WstxParsingException: Illegal to have multiple roots (start tag in epilog?).
 at [row,col,system-id]: [91,10,"file:/opt/hadoop-3.4.1/etc/hadoop/hdfs-site.xml"]
```

### 2.2 问题原因

`hdfs-site.xml` 文件中 `</configuration>` 标签位置错误，导致部分配置属性被放在了 XML 根元素外部。

**错误配置**:
```xml
</configuration>
<!-- NFS性能优化 -->
<property>
    <name>dfs.nfs3.stream.timeout</name>
    <value>3600000</value>
</property>
```

### 2.3 解决方法

将 NFS 配置属性移动到 `</configuration>` 标签内部，或删除多余的配置。

**修复后的配置**:
```xml
<property>
    <name>nfs.superuser</name>
    <value>*</value>
</property>
</configuration>
```

### 2.4 验证命令

```bash
# 验证 HDFS 配置是否正确
hdfs dfsadmin -report
```

## 3. YARN 端口绑定问题

### 3.1 问题现象

YARN ResourceManager (8088) 和 JobHistoryServer (19888) 只绑定到特定 IP (10.10.3.183)，无法通过 127.0.0.1 访问。

```bash
# 测试发现
Port 8088 (127.0.0.1): 000 无法访问
Port 8088 (10.10.3.183): 302 正常
```

### 3.2 问题原因

默认情况下，YARN 服务只绑定到配置的 hostname 对应的 IP 地址。

### 3.3 解决方法

#### 3.3.1 修改 yarn-site.xml

添加 ResourceManager 绑定配置：

```xml
<!-- 绑定 ResourceManager 到所有接口 -->
<property>
    <name>yarn.resourcemanager.bind-host</name>
    <value>0.0.0.0</value>
</property>
```

#### 3.3.2 修改 mapred-site.xml

添加 JobHistoryServer 绑定配置：

```xml
<!-- 绑定 JobHistoryServer 到所有接口 -->
<property>
    <name>mapreduce.jobhistory.bind-host</name>
    <value>0.0.0.0</value>
</property>
```

### 3.4 重启服务

```bash
# 停止服务
mapred --daemon stop historyserver
stop-yarn.sh

# 启动服务
start-yarn.sh
mapred --daemon start historyserver
```

### 3.5 验证命令

```bash
# 检查端口绑定
netstat -tlnp | grep -E "8088|19888"

# 测试本地访问
curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:8088
curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:19888
```

## 4. Grafana 端口绑定问题

### 4.1 问题现象

Grafana 默认只监听 IPv6 地址 (:::3000)，在某些环境下可能无法通过 IPv4 访问。

### 4.2 问题原因

Grafana 默认配置 `http_addr =` 为空，在某些系统上会默认监听 IPv6。

### 4.3 解决方法

修改 Grafana 配置文件，明确指定绑定到所有 IPv4 接口：

**文件**: `/opt/grafana-v12.0.2/conf/defaults.ini`

```ini
[server]
# 明确绑定到所有接口
http_addr = 0.0.0.0
http_port = 3000
```

### 4.4 重启 Grafana

```bash
# 停止 Grafana
sudo pkill -9 grafana

# 等待端口释放
sleep 2

# 启动 Grafana
sudo /opt/grafana-v12.0.2/bin/grafana server \
    --config=/opt/grafana-v12.0.2/conf/defaults.ini \
    --homepath=/opt/grafana-v12.0.2
```

### 4.5 验证命令

```bash
# 检查端口监听
netstat -tlnp | grep grafana

# 测试访问
curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:3000
curl -s -o /dev/null -w "%{http_code}" http://10.10.3.183:3000
```

## 5. 服务重启流程

### 5.1 完整重启所有服务

```bash
# 1. 停止所有服务
stop-yarn.sh
stop-dfs.sh
mapred --daemon stop historyserver
sudo pkill -9 grafana

# 2. 等待端口释放
sleep 3

# 3. 启动 HDFS
start-dfs.sh

# 4. 启动 YARN
start-yarn.sh

# 5. 启动 JobHistoryServer
mapred --daemon start historyserver

# 6. 启动 NFS Gateway
hdfs --daemon start nfs3

# 7. 重新挂载 HDFS NFS
sudo umount -lf /hdfs-nfs 2>/dev/null
sudo mount -t nfs -o nolock,soft,intr localhost:/ /hdfs-nfs

# 8. 启动 Grafana
sudo /opt/grafana-v12.0.2/bin/grafana server \
    --config=/opt/grafana-v12.0.2/conf/defaults.ini \
    --homepath=/opt/grafana-v12.0.2 > /tmp/grafana.log 2>&1 &
```

### 5.2 启动监控服务

```bash
# Prometheus (如果未运行)
/opt/prometheus/prometheus \
    --config.file=/opt/prometheus/prometheus.yml \
    --storage.tsdb.path=/opt/prometheus/data/ \
    --storage.tsdb.retention=15d \
    --web.enable-lifecycle &

# Loki (如果未运行)
/usr/local/bin/loki -config.file=/etc/loki/config.yaml &
```

## 6. 端口访问测试

### 6.1 一键测试脚本

```bash
#!/bin/bash
echo "=== 服务端口访问测试 ==="
echo ""

# 定义要测试的端口
ports=("3000" "8088" "19888" "9090" "3100")
names=("Grafana" "YARN RM" "JobHistory" "Prometheus" "Loki")

for i in ${!ports[@]}; do
    port=${ports[$i]}
    name=${names[$i]}
    
    echo -n "$name ($port): "
    
    # 测试本地访问
    local_code=$(curl -s -o /dev/null -w "%{http_code}" http://127.0.0.1:$port 2>/dev/null)
    if [ "$local_code" != "000" ] && [ -n "$local_code" ]; then
        echo -n "本地✅ "
    else
        echo -n "本地❌ "
    fi
    
    # 测试远程访问
    remote_code=$(curl -s -o /dev/null -w "%{http_code}" http://10.10.3.183:$port 2>/dev/null)
    if [ "$remote_code" != "000" ] && [ -n "$remote_code" ]; then
        echo "远程✅"
    else
        echo "远程❌"
    fi
done
```

| JobHistoryServer | 19888 | ✅ | ✅ |

## 8. 总结

本次运维工作的主要收获：

1. **XML 配置要谨慎**: 修改 Hadoop 配置文件时必须确保 XML 语法正确，特别是 `</configuration>` 标签的位置

2. **服务绑定要配置**: 默认情况下 Hadoop 和 Grafana 可能只绑定到特定 IP，需要显式配置 `bind-host` 或 `http_addr` 为 `0.0.0.0` 以允许所有接口访问

3. **验证不可少**: 每次修改配置后都要进行本地和远程访问测试，确保服务可以正常访问

4. **文档要记录**: 将问题和解决方案记录下来，避免重复踩坑
