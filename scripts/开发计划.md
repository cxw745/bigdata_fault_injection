# 大数据故障检测系统 - 开发计划

## 一、项目目标概述

本项目旨在构建一个完整的大数据故障检测系统，实现以下目标：

1. **数据收集**：从Hadoop集群收集系统指标、日志和拓扑信息
2. **故障注入**：支持多种故障类型的模拟和注入
3. **故障检测**：基于机器学习算法实现故障自动检测
4. **闭环系统**：形成完整的故障检测-注入-分析闭环

## 三、MVP核心组件

### 3.1 数据收集模块

```
collect_data/
├── collect_metrics.py    # Prometheus指标收集
├── collect_logs.py       # 日志收集
├── collect_topology.py   # 拓扑收集
└── collect_data.py       # 主入口
```

### 3.2 故障注入模块

```
故障类型（至少3种）:
├── data_bloat/          # 数据膨胀
├── data_skew/           # 数据倾斜
└── task_fail/           # 任务失败
```

### 3.3 故障检测模块

```
detection/
├── preprocess.py         # 数据预处理
├── feature_extraction.py # 特征提取
├── anomaly_detection.py  # 异常检测
├── classifier.py         # 故障分类
└── model_trainer.py     # 模型训练
```

### 3.4 NetLLM集成模块

```
netllm/
├── data_adapter.py       # 数据适配器
├── model_adapter.py      # 模型适配器
├── deployment.py         # 部署脚本
```

## 四、关键技术参数

### 4.1 数据收集参数

| 参数 | 值 | 说明 |
|------|------|------|
| 采集间隔 | 15秒 | Prometheus轮询间隔 |
| 指标类别 | CPU/内存/磁盘/网络 | 4大类系统指标 |
| 日志保留 | 5000条 | 单次查询限制 |
| 时间窗口 | 故障前3分钟+故障后5分钟 | 完整数据覆盖 |

### 4.2 故障注入参数

| 故障类型 | 参数 | 值 |
|----------|------|------|
| data_bloat | 膨胀倍数 | 4x |
| data_skew | 倾斜比例 | 80% |
| task_fail | 失败概率 | 20% |
| long_tail | 挂起时长 | 120秒 |
